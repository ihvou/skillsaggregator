# Tasks

> **Rule:** every task — new findings, follow-ups, deferred work, anything actionable — **must be added as a row in the table below**, not written as plain prose elsewhere in this file. If a task is too large for one row, split it into multiple rows. Keep the table the single source of truth.
>
> **Columns:** `ID | Priority | Status | Problem | Solution | Acceptance | Impact`
>
> **ID convention:** `B*` = Blocker, `M*` = Major, `MI*` = Minor, `N*` = Nit. IDs are immutable once assigned — when a task is done, mark it so in a `Status` column or strike through, but do not renumber.
>
> **Priority values:** `Blocker` (must fix before any deployment / demo recording) · `Major` (security, correctness, or visible spec deviation) · `Minor` (improvement, safe to defer) · `Nit` (polish).
